Hooks

Hooks 是 React@16.8新增的功能，Hooks 让你可以在函数组件中使用状态管理和 React 的其他特性。

Facebook 团队计划准备让 Hook 覆盖所有 class 组件的使用场景。

特点：

  • 在不改变组件层次结构的情况下重用有状态逻辑
  • Hook不能在 class 中使用，只能在函数组件使用
  • 只能在函数最外层使用 Hook，自定义 Hook 中调用 Hook
  • linter 插件用来检测 Hook API

自定义 Hook：如果函数的名字以 “use” 开头并调用其他 Hook，我们就说这是一个自定义 Hook

Hooks 的目的

1.很难去复用有状态组件

2.复杂的组件变得难以理解

3.Classes 混淆了人和机器

Hooks允许您在不改变组件层次结构的情况下重用有状态逻辑

BasicHooks

1.useState

一个状态值 和 一个更新状态的函数，初始化渲染时 state 为初始值initialState

setState 可以接收一个函数，该函数的参数为上一次状态值(setCount(prev=>prev+1))

useState 不会合并更新的对象

另一个选择是使用useReducer，更适合管理包含多个子值的状态对象

2.useEffect

Effect Hook 可以让你在函数组件中执行副作用操作，数据获取，设置订阅以及手动更改 React 组件中的 DOM 都属于副作用。

默认情况下，它在第一次渲染之后和每次更新之后都会执行，React 保证了每次运行 effect 的同时，DOM 都已经更新完毕。

传递给 useEffect 的函数在每次渲染中都会有所不同，这正是我们可以在 effect 中获取最新的 count 的值，而不用担心其过期的原因，每次我们重新渲染，都会生成新的 effect，替换掉之前的。某种意义上讲，effect 更像是渲染结果的一部分 —— 每个 effect “属于”一次特定的渲染

在 React 组件中有两种常见副作用操作：需要清除的和不需要清除的

this component sets the document title after React updates the DOM

使用 Hooks 的规则

1.只在顶层的使用 Hooks,不要在循环、条件或嵌套函数中使用

2.只在 React 函数组件和自定义 Hooks 中使用

可使用插件来规范 Hooks 的使用 - eslint-plugin-react-hooks

Custom Hooks

自定义 Hooks, 与功能相比更像是一种约定，函数名以 use 开头，且函数里使用了 Hooks，我们把这种称为自定义 Hooks。这种约定俗成的命名规则方便 eslint 插件来找到 Hooks 中的 bug
