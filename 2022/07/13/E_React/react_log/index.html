V16.1.0 — 2017.11.09

1.新版本不再发版到 bower

2.React.Children

React.Children 是顶层API之一，为处理 this.props.children 这个封闭的数据结构提供了有用的工具。

this.props 对象的属性与组件的属性一一对应，但是有一个例外，就是 this.props.children 属性。它表示组件的所有子节点。

V16.0.0 — 2017.09.26

1.不兼容<IE11

React 16依赖于 map 、 set 集合 以及 window.requestAnimationFrame(callback)，所以不再兼容<IE11，你可能需要使用 polyfill(Polyfill 是一块代码（通常是 Web 上的 JavaScript），用来为旧浏览器提供它没有原生支持的较新的功能。使旧的浏览器版本结合插件来模拟新版本浏览器的功能)

2.组件可以在 render 中return数组和字符串

3.提供ReactDOM.createPortal()功能

Portals 提供了一种很好的将子节点渲染到父组件以外的 DOM 节点的方式

语法ReactDOM.createPortal(vdom, dom)

4.renderToNodeStream和renderToStaticNodeStream

使用ReactDOMServer.renderToNodeStream()和ReactDOMServer.renderToStaticNodeStream()启用服务器端渲染的流模式。renderToString()仍然可以使用

虽然 facebook 并未使用 react 的服务器端渲染，缺少官方标准，目前为止解决方案层出不穷，next.js还是最佳方案。

5.vdom 支持非标准html属性

6.重构服务器端渲染

如果从服务器端接收到渲染的 html，使用ReactDOM.hydrate在浏览器端挂载，而不是ReactDOM.render，如果只是单纯的浏览器端渲染，继续使用ReactDOM.render挂载

V15.3.0 — 2016.07.30

1.增加React.PureComponent

一种适用于展示组件的组件，对于 state 进行浅比较，适用于不会多次改变的基础数据类型的 state，若是引用类型数据，则要引用不同才换更新 view。多次改变的基础数据类型 state，建议使用React.Component，在改变时调用shouldComponentUpdate()更新 view，浅比较也会消耗时间
